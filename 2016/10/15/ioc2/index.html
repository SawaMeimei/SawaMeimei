使用注解完成自己的一个DataBinding框架

发表于 2016-10-15 | 次阅读

需求场景

DataBinding,即将数据和view的行为绑定，在更新数据的时候自动更新view的显示。让coder更专注于业务上的数据变化。摆脱繁琐的setText,setBackground等等操作。

准备工作

以binding setText方法为例。将getData >> setData >> notifyUI 简化成 getData >> setData ，而notifyUI这一步放到和setData绑定，实现在setData 的时候自动notifyUI。

分解步骤为
1.将bean的字段和view的指定方法绑定（这里以setText为例
2.在setData的时候自动调用notifyUI的方法。
3.notifyUI的方法将自动获取bean的字段值和绑定的view的方法，并且调用绑定的view的方法传入值

实现代码

1.先定义一个注解，这个注解用来注解需要绑定的字段和绑定的对象，注解需要的数据有（绑定的方法名，绑定的方法的参数类型，绑定的view对象）

@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Bind {
    public int viewId();  //绑定的目标view的id
    public String methodName();   //绑定的view的方法
    public String policy();   //数据转化策略
    public Class methodParameter();   //绑定的view的方法的参数
}

注意到这里有个policy,数据转化策略，这个东西用来转化数据的类型。比如说，现在获取到一个String 类型的数据，但是我要在这个数据为null或“”的时候给对应的textview设置为”无”，又不希望改变数据的结构，这个策略就是用来将数据在设置到view之前的一个转化层。

2.注解了需要binnding的字段之后，就要获取到注解上的view和指定的字段绑定，这里以binding Activity中的view示例，提供一个统一的binding方法来将veiw和字段binding.


private HashMap<String,ArrayList<BindBlock>> $ = new HashMap<>();

public void bindWithPolicy(Activity activity) {
        try {
            //获取对象上所有的field并且尝试获取field上的Bind注解
            Field[] fields = this.getClass().getDeclaredFields();
            for (int i = 0; i < fields.length; i++) {
                Field valueField = fields[i];
                valueField.setAccessible(true);
                Bind annotation = valueField.getAnnotation(Bind.class);
                if (annotation != null) {
                    //如果注解存在，尝试获取注解上的信息
                    //注解上的信息有：binding对象的viewId，binding的方法名，binding的方法参数，数据转化策略。

                    //通过viewId获取到binding的view
                    int viewId = annotation.viewId();
                    View view = activity.findViewById(viewId);

                    //通过parameter和methodName获取到binding的method
                    Class parameter = annotation.methodParameter();
                    String methodName = annotation.methodName();
                    Method method = view.getClass().getMethod(methodName,parameter);
                    if(method==null){
                        throw new RuntimeException("Can not find method named"+methodName);
                    }

                    //获取数据转化策略，这里定义了一个IPolicy接口让自定义的数据转化策略实现
                    String policy = annotation.policy();
                    Object policyObj = null;
                    if(!TextUtils.isEmpty(policy)){
                        Class<?> policyClass = Class.forName(policy);
                        policyObj = policyClass.newInstance();
                        if(!(policyObj instanceof IPolicy)){
                            throw new RuntimeException("Incorrect policy");
                        }
                    }

                    //在binding的时候调用一次binding的method,给数据初始化到view上，这一步也是notifyUI的操作
                    Object value = valueField.get(this);
                    if(policyObj!=null){
                        value = ((IPolicy) policyObj).doPolicy(value);
                    }
                    method.invoke(view, value);

                    //把上面几个元素都存起来，当数据改变的时候用于notifyUI
                    ArrayList<BindBlock> arr = $.get(valueField.getName());
                    if(arr == null){
                        arr = new ArrayList<>();
                        //以binding的fieldName作为key
                        $.put(valueField.getName(),arr);
                    }
                    arr.add(new BindBlock(method, valueField, view, (IPolicy) policyObj));
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

这个BindBlock只是保存了几个元素的对象

public class BindBlock {

    public Method method;
    public Field fieldValue;
    public View bindView;
    public IPolicy policy;

    public BindBlock() {
    }

    public BindBlock(Method method, Field fieldValue, View bindView, IPolicy policy) {
        this.method = method;
        this.fieldValue = fieldValue;
        this.bindView = bindView;
        this.policy = policy;
    }

    public BindBlock(Method method, Field fieldValue, View bindView) {
        this.method = method;
        this.fieldValue = fieldValue;
        this.bindView = bindView;
    }
}

IPolicy

public interface IPolicy {
    public Object doPolicy(Object o);
}

通过binding方法之后，就将data和view 绑定了，当数据刷新的时候调用notifyUI方法，自动取出我们的method并调用invoke方法，就自动的把数据刷新到view上了

notifyUI方法

public void notifyUI() {
        try {
          //这里data改变的时候，只要找到指定的field绑定的method来调用invoke就行。
            StackTraceElement[] stack = Thread.currentThread().getStackTrace();
            String methodName = stack[3].getMethodName();
            String fieldName = methodName.replace("set", "");
            StringBuilder stringBuilder = new StringBuilder();
            fieldName = stringBuilder.append(Character.toLowerCase(fieldName.charAt(0))).append(fieldName.substring(1)).toString();
            ArrayList<BindBlock> arr = $.get(fieldName);

            //执行notifyUI的操作
            for(BindBlock block:arr){
                Object value = block.fieldValue.get(this);
                if(block.policy!=null){
                    value = block.policy.doPolicy(value);
                }
                block.method.invoke(block.bindView, value);
            }
        } catch (IllegalAccessException e) {
            e.printStackTrace();
        } catch (InvocationTargetException e) {
            e.printStackTrace();
        }
    }

现在来使用一下
先定义一个class来继承BindObtain,在需要binding的字段上加上@Bind注解

public class TestBean extends BindObtain {

    public String getTextContent() {
        return textContent;
    }

    public void setTextContent(String textContent) {
        this.textContent = textContent;
        notifyUI();
    }

    //@BindText(R.id.tv_content)
    @Bind(viewId = R.id.tv_content,policy = "com.example.hasee.mvvm.policy.TextPolicy",methodParameter = CharSequence.class,methodName = "setText")
    private String textContent;
}

注意到在set方法里除了数据的set还调用了notifyUI方法，这样在设置数据的同时就会刷新ui。

使用的数据转换策略是这样的

public class TextPolicy implements IPolicy {
    @Override
    public Object doPolicy(Object o) {
        if(o==null|| TextUtils.isEmpty(String.valueOf(o))){
            return "Nothing";
        }else{
            return String.valueOf(o);
        }
    }
}

public class MainActivity extends AppCompatActivity {

        super.onCreate(savedInstanceState);

        final TestBean testBean = new TestBean();

tips

在notifyUI方法里有这么一段代码

StackTraceElement[] stack = Thread.currentThread().getStackTrace();
String methodName = stack[3].getMethodName();
String fieldName = methodName.replace("set", "");
StringBuilder stringBuilder = new StringBuilder();
fieldName = stringBuilder.append(Character.toLowerCase(fieldName.charAt(0))).append(fieldName.substring(1)).toString();

这句话的代码是获取到notifyUI之前执行的入口方法的名字，在实例中都是在set方法执行的，通过字符串操作就可以获取是哪个field的set方法执行了，再通过fieldname就能获取到binding的几个元素。
也可以通过传入字符串的方式来获取。
关于StackTraceElement可以查看这篇文章

一些其他的问题

1.多个数据binding的情况，比如多个数据都binding到一个view，view的刷新是由多个数据共同决定。
比如bean中有三个String view显示其中最长的String，这种多个数据binding到一个view的情况还没办法解决。

2.这种binding形式持有了view的引用，数据要做好回收和释放，否则可能容易内存泄漏。

3.继承了BindObtain的bean不是单纯的POJO，不能通过gson直接转换成json字符串，也不能通过json字符串直接生成bean实体，需要另外想办法。

4.一个字段不能同时binding多个view的method，比如binding了setText就不能binding setVisible方法。
